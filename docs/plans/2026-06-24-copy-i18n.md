# Sprint: 文案 i18n（Western 平台输出英文）

> 承接 17a681d（跨境平台扩容）。新加的 reels(IG)/shorts(YT)/shopify(PDP) 是西方平台，却仍出中文文案——本轮让其出英文，抖音/小红书 保持中文。

## Phase 1: 需求分析（Think — CEO/产品视角）

### 背景
上轮激活 3 个 Western 平台并补全结构化 DNA，但文案引擎（COPY_SYSTEM + buildCopyPrompt）硬编码中文角色与"口语化中文"原则 → 跨境卖家选 IG/YT/Shopify 仍得到中文文案，名不副实。i18n 是上轮显式 defer 的收尾项，基建（preset 结构）已就绪。

### Scope（做什么）
- `platformPresets` 每平台加 `lang` 字段：抖音/小红书 = `"zh"`，reels/shorts/shopify = `"en"`。经 findPlatformPreset 取数（单源，不在 workflow.js 散落平台名分支）。
- 文案引擎按 `preset.lang` 分支（AI 主路径）：
  - `COPY_SYSTEM` 拆 zh/en 两版（en：英文爆款文案操盘手角色 + 反 AI-slop 英文版）。
  - `buildCopyPrompt` 注入**输出语言指令**（en → "Write all copy fields in natural, native English."），确保结构化指引即便中文，输出仍为目标语言。
- en 平台 `copyRules.ctaExamples` 改英文（单源同时修：规范示例 + 确定性兜底 CTA 都变英文）。

### Non-scope（不做什么）
- ❌ 确定性兜底 hook/caption 模板 i18n（buildVariants 中文连接词模板）→ defer；那是 AI 关闭/失败的降级路径，主路径优先。
- ❌ render*Guidance 全量本地化（框架描述/痛点推断 meta 指令）→ defer；输出语言指令已足以让模型出英文。
- ❌ UI/界面 i18n；多语言切换器；新增平台。
- ❌ 改 variants 数 / copy JSON 形状 / 「创作技能」门。

### Success（成功标准）
- preset.lang 经 findPlatformPreset 可取；无 lang 默认 zh（向后兼容）。
- en 平台：copy system 为英文版 + prompt 携带英文输出指令 + ctaExamples 英文；zh 平台：完全不变（中文 system + 中文指引）。
- 既有 zh 测试（框架/文案规范/痛点推断 等中文断言，跑在 小红书/抖音）0 回归。
- 新增测试：en 平台 prompt 走英文分支、zh 平台保持中文、lang 默认 zh。
- 全套测试 0 回归 + build ✓。

### Risks（风险）
- R1 散落硬编码：若在 workflow.js 写 `platform==="YouTube Shorts"` 判语言 → 破「平台单源」不变量。必须经 `preset.lang`。
- R2 既有 ai.test 中文断言跑在 zh 平台（小红书/抖音）→ 不受影响；但要确保 en 分支不误伤 zh。
- R3 并行会话仍在动仓库（components/db/supabase 未提交），domain.js 现干净但可能再被占用 → 提防，改完尽快提交。
- R4 lang 缺省/脏值 → 必须 default zh，不能让旧数据/Flow 合成 brief 走空。

→ 'go' 进入 Plan | 调整范围 | 'skip'

### 设计要点
- `platformPresets`：5 平台各加 `lang`（抖音/小红书=`"zh"`，reels/shorts/shopify=`"en"`）；en 三平台 `copyRules.ctaExamples` 改英文。
- `workflow.js`：
  - `COPY_SYSTEM` → `COPY_SYSTEM_ZH`（现状）+ `COPY_SYSTEM_EN`（英文操盘手角色 + 反 AI-slop 英文版）+ `copySystemFor(preset)`（`preset.lang==="en"?EN:ZH`，缺省 ZH）。
  - `buildCopyPrompt`：按 `preset.lang` 注入输出语言指令（en → "Write hook/caption/hashtags in natural native English"）；`lang!=="en"` 返回 ""（zh prompt 与现状逐字一致，向后兼容）。
  - 调用点 `system: COPY_SYSTEM` → `system: copySystemFor(preset)`。

## Phase 4: 审查结果（Review — 6 视角）

| 视角 | 结论 |
|------|------|
| 架构 | lang 作 preset 字段、copySystemFor/buildCopyPrompt 只读 `preset.lang`，零平台名分支，守「平台单源」 |
| 安全 | 无 secret / 无新输入路径；英文 system 同样含反 AI-slop |
| 性能 | 纯分支选择，无新网络/循环 |
| 代码质量 | COPY_SYSTEM_ZH/EN 对称；copySystemFor 缺省 zh 向后兼容；注释解释 WHY |
| 测试覆盖 | lang 契约 + en/zh 双向 system+prompt 断言；匹配 L2/L3 |
| 集成连续性（第6视角） | ① 继承不变量全过（单源/DNA/copyJSON/「创作技能」/variants/缺省zh）② 无 dead code：copySystemFor 被调用点用、langDirective 进 prompt ③ 既有 zh 中文断言（跑小红书/抖音）0 回归 ④⑤ 未碰并行会话文件 |
