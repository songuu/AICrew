# RoboNeo Studio 底部操作栏全面分析

> 设计参考文档。作为 AICrew「无限画布」实现（见 `docs/plans/2026-06-22-canvas-toolbar.md`）的需求与交互输入。

## 定位

这是一个**无限画布编辑器底部工具坞**，范式与 Figma / tldraw / Miro / Excalidraw 一致。
RoboNeo 把它拆成两个浮动 dock：中下「工具坞」（内容操作）+ 右下「视图坞」（视口操作）。

```
┌─────────────────────────────────────────────┐  ┌──────────────────────────┐
│  [选择] [抓手] [添加▾] │ [撤销] [重做]        │  │ − 25% +  [显示全部] [图层] │
│   工具组(模态互斥)        历史组(动作)          │  │  缩放      视口     面板     │
└─────────────────────────────────────────────┘  └──────────────────────────┘
        中下「工具坞」                                  右下「视图坞」
```

关键判断：**工具组是模态状态机（互斥单选）**，历史组/视图组是**瞬时动作**。截图中「抓手」高亮 = 当前激活态，证明 选择↔抓手 互斥切换。

## 逐控件拆解

### A. 工具组（模态，互斥 `activeTool`）

| 控件 | 图标 | 功能 | 交互模型 | 行业快捷键 | 状态 |
|------|------|------|----------|-----------|------|
| 选择 | 箭头 | 点选/移动/缩放对象 | 进入 select 模式，画布元素可命中 | `V` | 默认 |
| 抓手 | 手掌 | 平移视口，不命中元素 | 进入 pan 模式，拖拽=移动画布 | `H` / 长按`空格`临时切换 | 截图激活 |

工程要点：`activeTool` 单值状态机。**长按空格临时切 hand、松开回 select**（hold-to-pan）是专业画布的隐性约定，必备。

### B. 添加组（弹出菜单，非模态切换）

点击 `添加▾` 向上弹 popover，6 项插入源，分两类语义：

| 项 | 类型 | 行为 | 快捷键 |
|----|------|------|--------|
| 导入图片 | 外部资源 | 触发文件选择器 → 落点放置 | — |
| 导入视频 | 外部资源 | 同上（视频对象） | — |
| 文字 | 绘制图元 | armed，下次点击=插入文本框 | `T` |
| 矩形 | 绘制图元 | armed，下次拖拽=画矩形 | `R` |
| 圆形 | 绘制图元 | armed，下次拖拽=画椭圆 | `O` |
| 箭头 | 绘制图元 | armed，下次拖拽=画箭头/连线 | `A` |

两种语义不可混：**导入** = 异步取外部资源后放置；**图元** = "武装"一次性绘制工具（画完自动回 select）。

### C. 历史组（动作，依赖撤销栈）

| 控件 | 行为 | 禁用条件 | 快捷键 |
|------|------|----------|--------|
| 撤销 | 回退一步 | 撤销栈空（截图置灰） | `Ctrl/Cmd+Z` |
| 重做 | 前进一步 | 重做栈空 | `Ctrl/Cmd+Shift+Z` |

依赖一个 command/snapshot 栈。空栈必须 disabled。

### D. 视图坞（右下，独立 dock）

| 控件 | 行为 | 快捷键 |
|------|------|--------|
| `−` / `25%` / `+` | 缩放档位，百分比可点开下拉/重置 | `Ctrl±`、滚轮、双指捏合 |
| 显示全部 | zoom-to-fit，所有对象进视口 | `Shift+1` |
| 图层 | 切换图层面板（z-order / 显隐 / 锁定） | — |

缩放数字是**可交互入口**，不只是显示。视口操作与内容操作分离成两个 dock 是刻意设计。

## 交互状态机（实现蓝图）

- **工具态**：`tool ∈ {select, hand, rect, ellipse, text, arrow}`，单值；draw 工具画完回 select。
- **手势态（瞬时，不入历史）**：pan（视口平移）、move（移动对象）、draw（绘制新图元）。
- **历史**：仅对象变更入栈（add/move/update/delete/reorder）；**视口 pan/zoom 不进历史**（与 Figma 一致）。
- **每手势一条历史**：拖拽期间用 draft 临时态渲染，pointerup 才 commit 一条 → 一次 undo 撤销整个手势。

## 对照 AICrew 现状（差距分析，分析时点）

AICrew 已有同款外观（`src/AICrewStudio.jsx` `bottom-tool-dock` + `zoom-dock`），但**是皮肤不是功能**：

| 控件 | RoboNeo（真实） | AICrew 分析时现状 | 差距 |
|------|----------------|-------------------|------|
| 选择 | 画布选择模式 | `navigate("dashboard")` 路由跳转 | 借壳做导航 |
| 抓手 | 平移视口 | `navigate("workbench")` | 同上 |
| 添加▾ | 6 项插入 popover | 单跳，无菜单 | 缺 popover + 图元系统 |
| 撤销/重做 | 撤销栈 | `disabled` 死按钮 | 无历史栈 |
| 视图坞 | 缩放/适应/图层 | 显示 agents/credits | 改成状态展示 |

根因：AICrew 是 `view`-switch 多视图静态站，**没有真正的画布运行时**（无 canvas、无对象模型、无视口变换、无命令栈）。

## 落地结论

复刻需 4 个子系统：① 画布运行时（对象模型 + 视口变换 + 命中测试）② 工具状态机 ③ 命令栈 ④ 图层面板。
静态导出契约（`output:export`/`basePath:/aicrew`）**不阻碍**——画布是纯客户端 React，无需后端。
实现见 sprint：`docs/plans/2026-06-22-canvas-toolbar.md`。
